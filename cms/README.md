# DOLPHLINK CMS

This folder is your **content management system**. Edit the CSV files
here in Excel / Numbers / Google Sheets / VS Code, then run `build.py`
to regenerate `content/content.json` (which the live site reads).

---

## Files

| File | Rows | What it controls |
| --- | --- | --- |
| `text.csv` | ~35 | All single-value strings (hero copy, video caption, footer, etc.) — flat `key,value` |
| `menu.csv` | any | Top nav menu (`label, scrollTo, href, target`) — add/remove rows freely |
| `stats.csv` | 4 | Reliability Matrix cards (`key, icon, value, label, desc`) |
| `portfolios.csv` | 9 | Portfolio buttons (`key, icon, label, tagline, desc`) |
| `audit.csv` | 4 | Security Audit boxes (`title, desc`) |
| `sectors.csv` | 9 | Standardization Ecosystem nodes (`name, color, desc`) |
| `locations.csv` | 12 | Global map points (`name, lng, lat, isHQ`) |

### menu.csv specifics

Each row is one nav link. Two flavors:

* **In-page anchor** — set `scrollTo` to a section id (e.g. `n-sec`). Leave `href` and `target` blank. Click smooth-scrolls to that section.
* **External link** — set `href` to a full URL. Set `target=_blank` to open in a new tab. Leave `scrollTo` blank.

If both `scrollTo` and `href` are filled, `scrollTo` wins.

Available section ids in current HTML: `h-sec` (Hero), `n-sec` (Industrial Portfolios), `a-sec` (Security Audit). To add a new in-page link you must also add a `<section id="...">` with that id to `index.html`.

## Quick start

```bash
# After editing any CSV:
python cms/build.py
```

Output:

```
✓ Wrote content/content.json (2812 bytes)
  Reload your browser — Live Server picks up the change automatically.
```

If something is wrong (typo, missing column, bad number) the script
will refuse to overwrite the old JSON and tell you exactly which row
broke. Your site stays alive.

### Sanity check without writing

```bash
python cms/build.py --check
```

## Editing rules

1. **Don't change the column header row** — the script matches by
   header name. Adding a NEW column is fine and ignored.
2. **Quotes around commas**: if a value contains a `,` you must wrap
   the cell in `"..."`. Excel and Google Sheets do this automatically.
3. **Save as UTF-8**. Excel: *Save As → CSV UTF-8*. Numbers: *Export
   To → CSV → Encoding UTF-8*. The script also tolerates the BOM that
   Excel adds.
4. **`isHQ` column** in `locations.csv` accepts: `true / false / yes /
   no / 1 / 0` (case-insensitive).
5. **Coordinates** in `locations.csv` are `lng, lat` (longitude
   first), in decimal degrees. East / North are positive.
6. **Icon names** in `stats.csv` and `portfolios.csv` must match
   filenames in `media/icon/3D/*.webp` without extension. Available
   icons: `resilience, territories, velocity, ai-h, solution,
   touch, oneAPI, cross, smartlink, combine, dashboard, phone, logo`.
7. **Icon names** are case-sensitive on Linux. `oneAPI` is correct,
   `oneapi` is not.

## Adding / removing rows

Just add or delete rows in the CSV — the page automatically shows
however many cards / sectors / locations you have. The CSS grid
flexes to fit.

* Want a 5th stat card? Add a row to `stats.csv`. (Note: layout looks
  best with 4. >4 will wrap on mobile.)
* Want to remove a city from the map? Delete that row in
  `locations.csv`.
* Want to add a 10th sector? Add it to `sectors.csv`. Eco-graph
  recomputes node positions automatically.

---

## Don't edit `content/content.json` by hand

It's overwritten every time `build.py` runs. The auto-generated file
even tells you that:

```json
"_comment": "Auto-generated by cms/build.py from the CSVs in /cms. DO NOT edit by hand — edit the CSVs and rerun: python cms/build.py"
```
